**Para qualquer copywriter falando sobre qualquer produto Anova.**
Este documento é o contrato editorial. Se o texto que você escreveu contradisser qualquer regra aqui, o texto perde.

## 01 · Identidade — uma frase

**Anova é o Sistema Operacional Conversacional de Conexão de Negócios do mercado financeiro brasileiro.**

Categoria nova. Não é CRM. Não é chatbot. Não é app de investimento. É a camada operacional onde research, carteira, conversa e execução viram um fluxo só — dentro do WhatsApp, com 80+ instituições reguladas.

**Tagline:** _Confiança que vira execução._

**Anti-tagline** (o que a Anova NÃO é):
- Não é "mais uma plataforma"
- Não é "CRM com IA"
- Não é "bot de atendimento"
- Não é "app de investimento"
- Não é "fintech de produto único"

## 02 · A narrativa — 7 capítulos em 1 minuto

Toda copy da Anova deve conseguir se ancorar em pelo menos um dos 7 capítulos abaixo. Se o texto não conecta com nenhum, ele está fora da história.

| # | Capítulo | Dor que resolve | Dor que criou |
|---|----------|-----------------|---------------|
| 01 | **Research** — casa de análise | Análise descritiva não antecipa | Cliente quer execução, não newsletter |
| 02 | **Gestão Patrimonial** — carteiras automáticas | Tese não vira ordem | Rede não certificada trava crescimento |
| 03 | **Rede Própria** (Wealth Hubs) | Rede não certificada | Proposta manual não escala |
| 04 | **Life Invest** — algoritmo de proposta | Proposta manual | Volume monoproduto explode |
| 05 | **Rede Monoproduto** — crédito, consórcio, seguro | Produto único pulverizado | Atendimento sem governança |
| 06 | **Orquestrador** — camada conversacional | Atendimento caótico no WhatsApp | Falta visão de sistema |
| 07 | **Anova OS** — infraestrutura | Mercado sofre a mesma dor | Conexão de negócios vira sistema |

**Regra:** cada produto é o remédio da dor do anterior — e nasceu da dor que o próximo resolve. Use isso como ponte entre páginas.

## 03 · Framework primário — Golden Circle

Todo texto de produto Anova segue Sinek: **Por que → Como → O que.**

### Para página de produto

**WHY — a crença**
Abertura (hero / § 00): Começar com uma crença sobre o mercado, não com uma feature.
Exemplo: _"Acreditamos que confiança só vira resultado quando vira sistema."_

**HOW — o método**
Meio (virada / § 02): A resposta concreta de como a Anova opera isso de forma diferente.
Exemplo: _"Multi-custódia real. HITL em ações sensíveis. Dentro do WhatsApp, não sobre."_

**WHAT — a superfície**
Produtos, features, números (§ 03 em diante).
Exemplo: _"6 superfícies. 80+ instituições. R$ 2Bi sob operação."_

### Regra prática

Se a hero fala do **produto** antes da crença → a copy está errada.
Se o texto explica a **feature** antes do método → a copy está errada.
Quem não acredita no WHY não compra o WHAT.

## 04 · Framework complementar — Flywheel

Cada superfície resolve a dor da anterior. Isso é a prova de que a Anova virou infraestrutura — cresceu por dentro, não por roadmap de slide.

**Use Flywheel quando:**
- Quiser explicar por que a Anova é diferente
- Quiser justificar por que o pacote inteiro supera a soma das partes
- Quiser mostrar que a evolução foi forçada pelo mercado, não pelo marketing

**Não use Flywheel quando:**
- Estiver vendendo uma superfície isolada
- Estiver respondendo a uma pergunta simples ("quanto custa?", "como funciona?")

## 05 · Voz e tom — editorial, não corporativo

A Anova escreve como manifesto, não como brochure. Isso tem consequências práticas:

### Rules

1. **Primeira palavra em bold** como âncora visual. Ex: _"**Acreditamos** que..."_ / _"**Seis** superfícies."_
2. **Chapter markers** (§ 00, § 01, § 02 etc) — estrutura editorial explícita. Sempre.
3. **Parágrafos curtos.** Ponto final conta frase. Três frases por parágrafo é o teto.
4. **Frases com ritmo de manifesto.** _"Não é CRM. Não é bot. É o sistema operacional de uma rede que ainda não existia."_
5. **Afirmação antes da qualificação.** _"Dentro do WhatsApp."_ Não _"Disponível dentro do WhatsApp."_
6. **Evitar enumeração preguiçosa.** Em vez de _"A, B, C e D"_ — reescrever com ritmo.
7. **Nomes próprios dos produtos em Title Case.** Research, Life Invest, Orquestrador, Gestão Patrimonial, Wealth Hubs, Crédito.

### Banidos — nunca escreva

- "Revolucionário", "inovador", "disruptivo"
- "Solução completa", "plataforma integrada"
- "No fim do dia", "de ponta a ponta"
- "Robusto", "escalável", "flexível"
- Exclamações (!) — nunca, em nenhum lugar
- Emojis — nunca
- Adjetivos superlativos sem número por trás

### Permitidos — use com intenção

- "Camada" (vs "plataforma")
- "Superfície" (vs "produto" ou "ferramenta")
- "Flywheel" (no contexto narrativo)
- "Multi-custódia", "HITL", "audit-by-design" — vocabulário técnico funciona porque é específico
- "Artesanal" — quando falar do problema (distribuição artesanal)
- "Infraestrutura" — quando falar da solução

## 06 · Vocabulário crítico

### Distribuição vs. Conexão de negócios

- **"Distribuição" = problema.** Artesanal, pulverizada, manual.
- **"Conexão de negócios" = solução.** Sistema, auditável, infraestrutura.

Use o primeiro para descrever o estado antigo. Use o segundo para descrever o que a Anova entrega.

### Chatbot vs. Orquestrador

- "Chatbot" é o que os outros fazem. Responde pergunta. Morre no input.
- "Orquestrador" é o que a Anova faz. Governa decisão. Tem login regulado, trilha imutável, HITL.

Nunca confundir. Nunca chamar o Orquestrador de "bot".

### CRM vs. Sistema Operacional

- CRM é cadastro. Aceita dado.
- Sistema operacional é ação. Executa decisão.

A Anova faz System of Action + System of Record no mesmo lugar.

### App vs. Camada

- App pede migração. Pede download. Pede atenção nova.
- Camada vai onde o cliente já está (WhatsApp, API oficial Meta).

A Anova é camada. Nunca app.

## 07 · Estrutura canônica de página de produto

Todas as páginas (exceto index) seguem este esqueleto. Adapte números de seção conforme necessário, mas mantenha a sequência.

```
§ 00 · Hero — WHY
     H1: crença + produto como resposta
     Sub: método (HOW) + prova
     CTA: verbo + benefício específico

§ 01 · Problema
     O que está quebrado hoje, na linguagem de quem vive
     Fricções concretas, não conceituais

§ 02 · A virada — HOW
     Como a Anova opera diferente
     3-4 pontos de método, cada um com evidência

§ 03 · O que é — WHAT
     Features traduzidas em capacidades
     Imagens do produto / UI real
     Listas factuais (não adjetivadas)

§ 04 · Como funciona — fluxo operacional
     3-5 passos com nomes próprios
     Links para superfícies conectadas

§ 05 · Prova social
     Depoimentos, números, mídia

§ 06 · Diferencial competitivo
     Tabela comparativa OU
     Por que ninguém mais consegue (moat)

§ 07 · Para quem
     Segmentos claros com CTA próprio

§ 08 · Próximo passo — CTA final
     Agendar, licenciar, ser credenciado — conforme a superfície
```

## 08 · Ângulo por produto

Cada superfície tem um ângulo específico. Respeite-o — não misture.

### Gestão Patrimonial (`wealth-management.html`) — **B2C · investidor final**

⚠️ **Esta é a única página B2C do site.** Todas as outras falam com operadores/empresas. Aqui você fala com o investidor PF — pessoa que tem patrimônio e vai ser atendida. **Nunca** use "opere", "método", "sua operação", "como operamos" — isso é linguagem de operador. Use "seu patrimônio", "sua carteira", "você", "a gente te atende".

- **Audience:** Investidor PF qualificado cansado de rebate. Médico, advogado, engenheiro, empresário, executivo, herdeiro. Já foi atendido por assessor/banker e percebeu o modelo.
- **WHY-hook (cliente-facing):** _"Acreditamos que o seu patrimônio merece uma estrutura que só ganha quando ele ganha."_
- **HOW-pontos (em linguagem do cliente):** Seu dinheiro fica na sua conta. A gente só cresce quando sua carteira cresce. Sem rebate, sem spread, sem produto empurrado. Research nossa alimentando sua carteira.
- **WHAT-âncora (traduzida pro cliente):** O modelo das famílias ultra-ricas — traduzido em software e acessível pra você.
- **CTA:** "Falar com um time de gestão patrimonial" / "Solicitar proposta de alocação" / "Marcar diagnóstico de patrimônio"
- **Conexão com outras páginas (texto-âncora cliente-facing):**
  - Research → _"ler a Research que alimenta sua carteira"_
  - Life Invest → _"como sua carteira é montada e rebalanceada"_ (nunca chamar de "Life Invest" sem contextualizar o benefício)
  - Orquestrador → _"o canal de atendimento pelo WhatsApp"_ (nunca chamar de "Orquestrador" sem contextualizar)
  - Crédito → _"acessar crédito multibanco pela mesma estrutura"_

**Seções a serem nomeadas com cuidado:**
- Cards da virada: usar "PARA VOCÊ · 01/02/03..." ou "VOCÊ RECEBE · 01..." — nunca "MÉTODO" ou "SOLUÇÃO" (muito técnico).
- Eyebrows: "Como cuidamos do seu patrimônio" (não "como operamos").

### Wealth Hubs (`wealth-hubs.html`)
- **Audience:** Gestora licenciada, escritório que quer montar gestora, hub de assessores
- **WHY-hook:** _"Acreditamos que montar gestora hoje não deveria exigir reconstruir infraestrutura."_
- **HOW-pontos:** Stack inteira embarcada, backoffice + compliance, 80+ parceiros, Research + Life Invest + Orquestrador
- **WHAT-âncora:** Sua marca na ponta. O peso regulatório e tecnológico com a gente.
- **CTA:** "Ver modelo de licenciamento" → formulário
- **Conexão com outras páginas:** todas — Wealth Hubs É o pacote completo

### Orquestrador (`orquestrador.html`)
- **Audience:** Diretor de operações, sócio de rede, compliance
- **WHY-hook:** _"Acreditamos que governança não é departamento — é arquitetura."_
- **HOW-pontos:** API oficial WhatsApp Business, login regulado, trilha imutável, HITL, 6 verticais reguladas
- **WHAT-âncora:** Sistema operacional conversacional de conexão de negócios — não chatbot
- **CTA:** "Ver o Orquestrador em ação"
- **Conexão com outras páginas:** Life Invest (carteira entra pelo canal), Crédito (cotação multibanco pelo canal), Gestão Patrimonial (relacionamento pelo canal)

### Life Invest (`life-invest.html`)
- **Audience:** Consultor, assessor, rede de representantes
- **WHY-hook:** _"Acreditamos que proposta de carteira não deveria ser artesanato de PDF."_
- **HOW-pontos:** Áudio em → PDF em 5 min, 50k simulações Monte Carlo, consulta multibanco em tempo real, três cenários
- **WHAT-âncora:** Áudio vira carteira institucional em 5 minutos
- **CTA:** "Ver Life Invest rodando"
- **Conexão com outras páginas:** Research (teses ativas), Gestão Patrimonial (execução), Orquestrador (entrega)

### Crédito (`credito.html`)
- **Audience:** Originador, rede monoproduto, cliente que quer financiamento
- **WHY-hook:** _"Acreditamos que cotação de crédito deveria ser do lado do cliente — não do banco."_
- **HOW-pontos:** 80+ instituições cotando a mesma operação, grupo WhatsApp originador + cliente + especialista, comparação por critério do cliente
- **WHAT-âncora:** Financiamento, consórcio, home equity, crédito empresarial, seguro, saúde — no mesmo canal
- **CTA:** "Cotar crédito multibanco"
- **Conexão com outras páginas:** Orquestrador (o grupo roda dentro dele)

### Research (`research.html`)
- **Audience:** Mesa, consultor, investidor qualificado, imprensa financeira
- **WHY-hook:** _"Acreditamos que descritivo é passado. Decisão vem de preditivo."_
- **HOW-pontos:** Wyckoff reescrito em software, correlações cross-asset, volume profile, calibração contínua
- **WHAT-âncora:** Antecipou Argentina 2023, Chile + Colômbia 2025, dólar e Selic 2025
- **CTA:** "Receber a Research" / "Ler o último call"
- **Conexão com outras páginas:** alimenta Life Invest, Gestão Patrimonial e Wealth Hubs

## 09 · Fórmulas de headline

Escolha por necessidade, não por gosto. Cada fórmula tem uso específico.

### Fórmula A · Crença (hero de produto)
**"Acreditamos que [X] não deveria ser [Y]."**
Uso: quando a página introduz a crença fundadora.
Exemplo: _Acreditamos que confiança só vira resultado quando vira sistema._

### Fórmula B · Contraste (hero alternativo)
**"[Palavra-problema] é [estado-antigo]. [Palavra-solução] é [estado-novo]."**
Uso: quando a categoria precisa ser contrastada contra o que existia.
Exemplo: _Distribuição é artesanato. Conexão de negócios é sistema._

### Fórmula C · Ação (hero de conversão)
**"[Verbo imperativo] como [X]. Não como [Y]."**
Uso: quando a página vende método / operação.
Exemplo: _Opere como sistema. Não como pilha de ferramentas._

### Fórmula D · Número (seção de superfície)
**"[N] [substantivo plural]. [Afirmação unificadora]."**
Uso: abertura de seção que apresenta múltiplos pontos.
Exemplo: _Seis superfícies. Uma infraestrutura só._

### Fórmula E · Consequência (problema / pain)
**"[Estado-antigo] é indefensável. [Quem paga]."**
Uso: abertura da seção de problema.
Exemplo: _Distribuição pulverizada é indefensável. O time paga a conta._

## 10 · Fórmulas de subheadline / lead

### Sub A · Por isso existe
_"Por isso existe a [produto]: [método]. [Prova concreta]."_

### Sub B · Método explícito
_"A [produto] é a [substantivo que não é ferramenta] onde [X, Y, Z] viram [resultado novo]."_

### Sub C · Prova antes da explicação
_"[Número]. [Adjetivo]. [Verbo]. A [produto] [descrição]."_

## 13 · Âncoras de conexão entre páginas

Toda página Anova linka para pelo menos 2 outras superfícies. Isso reforça que é uma camada só. Use sempre os paths:

- `research.html` → Research
- `life-invest.html` → Life Invest
- `orquestrador.html` → Orquestrador
- `wealth-management.html` → Gestão Patrimonial
- `wealth-hubs.html` → Wealth Hubs
- `credito.html` → Crédito

**Regra:** links contextuais ficam inline no texto (não em listas soltas), com texto âncora que explica o que a página linkada faz.

✅ Bom: _"A tese vem da [Research](research.html), processa-se no [Life Invest](life-invest.html) e executa pelo [Orquestrador](orquestrador.html)."_

❌ Ruim: _"Saiba mais sobre Research aqui."_

## 14 · Checklist antes de publicar qualquer copy

```
[ ] O hero tem uma crença (WHY) — não uma feature
[ ] Há chapter marker § em cada seção
[ ] Primeira palavra da H1 está em bold
[ ] Todo produto nomeado linka para sua página
[ ] Palavra "chatbot" não aparece
[ ] Palavra "distribuição financeira" só aparece no contexto de problema
[ ] Toda afirmação numérica tem origem na lista autorizada
[ ] Frases curtas — três por parágrafo, no máximo
[ ] Zero exclamações, zero emojis
[ ] CTA é verbo + benefício específico
[ ] Há pelo menos 2 links inline para outras superfícies
[ ] A página poderia ser lida de pé, em 90 segundos, e fazer sentido
```

Se qualquer linha falhar — reescreva antes de publicar.

## 15 · Quando estiver em dúvida

Abra o `NARRATIVE.md`. Procure o capítulo da superfície que você está escrevendo. Reescreva a frase amarrando-a ao momento do flywheel.

Se o texto não consegue se ancorar em nenhum capítulo do NARRATIVE, o texto está fora da Anova.

**O contrato editorial da Anova é este:** toda copy deve fazer o leitor entender _por que_ esta camada existe, _como_ ela opera diferente e _o que_ ela entrega — nessa ordem, com essa especificidade, sem brochura.
